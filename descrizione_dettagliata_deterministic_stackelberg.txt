DESCRIZIONE DETTAGLIATA - deterministic_stackelberg.py
=======================================================

STATO ATTUALE DEL CODICE
========================

Il file implementa un modello epidemiologico deterministico SIRS/SIR con
Stackelberg game:
- leader: il governo, che sceglie la spesa di controllo c_s;
- follower: i cittadini, che rispondono con una socialita x_t^*;
- dinamica epidemiologica: compartimenti S, I, H, R, D con ospedalizzazione,
  decessi ospedalieri e reinfezione opzionale.

La run principale non esegue piu una singola simulazione: confronta due profili
di comportamento dei cittadini, uno con best response quadratica e uno con
best response logaritmica, poi stampa le metriche comparative e salva quattro
grafici PNG.

Le funzioni di scansione e calibrazione restano nel file come strumenti di
analisi, ma non fanno parte del flusso standard richiamato da main().


MODELLO E FORMULE USATE NEL CODICE
==================================

1) Prescrizione sociale del governo
------------------------------------
x_bar(c_s) = 1 / (1 + kappa * c_s)

Nel file esiste anche la variante esponenziale della mappatura, ma nella
simulazione principale viene usata quella logistica.

2) Rischio percepito
--------------------
p_t = clip(rho_rischio * (I_t + H_t) / N, 0, 1)

Il rischio percepito dipende da infetti e ospedalizzati, non dai soli infetti.

3) Best response dei cittadini
------------------------------
3.1 Quadratica
x_t^* = clip(x_bar - (lambda_rischio * p_t) / eta_compliance, 0, 1)

3.2 Logaritmica pura
U(x) = a * log(x + epsilon) - lambda_rischio * p_t * x - (eta / 2) * (x - x_bar)^2

La best response logaritmica viene trovata numericamente su griglia.

4) Fattore di contatto sociale
------------------------------
new_infections_t = f(x_t^*) * beta * S_t * I_t / N
f(x) = x^potenza, con default potenza = 1.0

5) Dinamica epidemiologica
--------------------------
S_{t+1} = S_t - new_infections_t + new_waning_immunity_t
I_{t+1} = I_t + new_infections_t - new_hospitalizations_t - gamma * I_t
H_{t+1} = H_t + new_hospitalizations_t - gamma_h * H_t - delta_h * H_t
R_{t+1} = R_t + gamma * I_t + gamma_h * H_t - new_waning_immunity_t
D_{t+1} = D_t + delta_h * H_t

con:
- new_hospitalizations_t = h * new_infections_{t-tau}
- gamma_h = 1 / degenza_media_ospedaliera
- delta_h = tasso_decesso_ospedaliero
- new_waning_immunity_t = min(omega * R_t, R_t) se le reinfezioni sono abilitate

6) Costo del governo su finestra di previsione
----------------------------------------------
J(c_s) = J_epidemico + J_controllo

J_epidemico somma nel tempo:
C_t = c_i * I_t + k * (exp(alpha * (H_t / capacita_ospedaliera)) - 1)

J_controllo:
lambda_reg_controllo * c_s^2 * (orizzonte + 1)

Il controllo entra quindi in modo diretto tramite la penalizzazione quadratica
e in modo indiretto modificando la traiettoria prevista di I_t e H_t.

FLUSSO RUN PRINCIPALE (EFFETTIVO)
=================================

Il blocco finale del file usa if __name__ == "__main__" e chiama main().
La run principale:

1) stampa intestazione simulazione e stato delle reinfezioni
2) esegue una simulazione quadratica completa con MPC periodico
3) esegue una simulazione logaritmica completa con MPC periodico
4) calcola e stampa le metriche comparative:
   - picco infetti e giorno del picco
   - picco ospedalizzati e giorno del picco ospedalizzati
   - decessi cumulati finali
   - costo totale epidemico con controllo variabile
   - compliance media x*/x_bar
   - gap medio x* - x_bar
5) stampa i delta logaritmica - quadratica
6) genera i grafici e li salva in PNG
7) mostra i grafici se mostra_grafici_default = True
