DESCRIZIONE DETTAGLIATA - deterministic_stackelberg.py
======================================================

STATO ATTUALE DEL CODICE
========================

Il file implementa un modello epidemiologico deterministico Stackelberg con:
- leader: governo, che decide una spesa di controllo c_s;
- follower: cittadini, che rispondono con socialita effettiva x_star;
- dinamica epidemiologica S-I-H-R-D con ritardi espliciti sui flussi;
- reinfezioni opzionali (R -> S) tramite ritardo fisso tau_IRS.

La run principale non esegue una sola simulazione, ma confronta due specifiche
di utilita dei follower:
- best response quadratica;
- best response logaritmica (massimizzazione numerica su griglia).

Alla fine stampa metriche comparative e salva quattro grafici PNG.


MODELLO E FORMULE ATTUALI
=========================

1) Prescrizione sociale dal controllo pubblico
----------------------------------------------
x_bar(c_s) = 1 / (1 + kappa * c_s)  (default usata: logistica)

E presente anche la variante esponenziale exp(-kappa * c_s), ma nel flusso
operativo viene usata la forma logistica.

2) Rischio percepito
--------------------
p_t = clip(rho_rischio * (I_t + H_t) / N, 0, 1)

Il rischio dipende da infetti non ospedalizzati + ospedalizzati.

3) Best response dei cittadini
------------------------------
3.1 Quadratica:
x_star = clip(x_bar - (lambda_rischio * p_t) / eta_compliance, 0, 1)

3.2 Logaritmica con compliance:
U(x) = a * log(x + epsilon)
       - lambda_rischio * p_t * x
       - (eta_compliance / 2) * (x - x_bar)^2

La best response logaritmica e trovata su griglia [0, 1].

4) Termine di contagio
----------------------
f(x_star) = x_star^potenza, con default potenza = 1.0

new_infections_t = f(x_star_t) * beta * S_t * I_t / N

5) Dinamica compartimentale con ritardi (implementazione corrente)
------------------------------------------------------------------
Flussi costruiti su code storiche:
- new_hospitalizations_t = h * new_infections_{t - tau_IH}
- new_recoveries_non_h_t = (1 - h) * new_infections_{t - tau_IR}
- new_recoveries_h_t = (1 - mu_H) * new_hospitalizations_{t - tau_HR}
- new_deaths_h_t = mu_H * new_hospitalizations_{t - tau_HD}
- new_waning_immunity_t = ingressi_in_R_{t - tau_IRS} (se reinfezioni abilitate)

Aggiornamento stati:
- S_{t+1} = S_t - new_infections_t + new_waning_immunity_t
- I_{t+1} = max(0, I_t + new_infections_t - new_hospitalizations_t - new_recoveries_non_h_t)
- H_{t+1} = max(0, H_t + new_hospitalizations_t - new_recoveries_h_t - new_deaths_h_t)
- R_{t+1} = max(0, R_t + new_recoveries_non_h_t + new_recoveries_h_t - new_waning_immunity_t)
- D_{t+1} = max(0, D_t + new_deaths_h_t)

6) Costo del governo su finestra MPC
------------------------------------
J(c_s) = J_epidemico + J_controllo

J_epidemico usa costo istantaneo con:
- termine lineare su I_t: c_i * I_t
- termine di saturazione ospedaliera su H_t:
  k * (exp(alpha * (H_t / capacita_ospedaliera)) - 1), con clipping numerico.

J_controllo = lambda_reg_controllo * c_s^2 * (orizzonte + 1)

Il governo ottimizza c_s su griglia in [c_min, c_max].

FLUSSO PRINCIPALE EFFETTIVO
===========================

Il blocco finale usa if __name__ == "__main__" e chiama main().

main() esegue:
1) stampa intestazione simulazione e stato reinfezioni;
2) simulazione completa con best response quadratica;
3) simulazione completa con best response logaritmica;
4) calcolo e stampa metriche comparative:
   - picco infetti e giorno;
   - picco ospedalizzati e giorno;
   - decessi finali cumulati;
   - costo epidemico totale (con controllo variabile);
   - complianza media x_star/x_bar;
   - gap medio x_star - x_bar;
5) stampa dei delta (logaritmica - quadratica);
6) generazione e salvataggio grafici;
7) visualizzazione grafici se mostra_grafici_default = True.
