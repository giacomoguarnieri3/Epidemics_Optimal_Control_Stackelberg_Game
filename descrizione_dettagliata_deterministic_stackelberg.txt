DESCRIZIONE DETTAGLIATA - deterministic_stackelberg.py
=======================================================

STATO ATTUALE DEL CODICE
========================

Il file implementa un modello SIR deterministico con struttura Stackelberg:
- Leader: Governo, che sceglie una intensita di controllo c_s.
- Follower: Cittadini, che scelgono socialita x_t^* in risposta a prescrizione e rischio.
- Dinamica: SIR discreta, con contagio modulato dalla socialita scelta.

Stato operativo attuale (main):
- In una singola run vengono eseguite SOLO due simulazioni complete:
  1) utility cittadino quadratica
  2) utility cittadino logaritmica
- Il codice stampa risultati di entrambe e i delta di confronto.
- Produce quattro grafici PNG (2 per quadratica, 2 per logaritmica) e mostra le figure.
- Le funzioni di scansione/calibrazione restano nel file ma NON sono chiamate nel flusso principale.

Organizzazione attuale del file (riordino leggibilita):
- 1) Import librerie
- 2) Parametri del modello
- 3) Layer follower: comportamento cittadini
  - 3A) Mappatura policy/prescrizione + utility
  - 3B) Best response per specifica di utility
  - 3C) Utility numeriche di supporto
- 4) Funzioni di costo epidemiologico
- 5) Dinamica Stackelberg e ottimizzazione del leader
  - 5A) Simulazione/costo su finestra
  - 5B) Simulazione completa MPC
- 6) Analisi output
  - 6A) Visualizzazione
  - 6B) Scansioni e calibrazioni parametriche
- 7) Script di esecuzione (entrypoint)

Regola di ordinamento adottata nei sottogruppi:
- Prima funzioni helper (mattoni di supporto/riuso)
- Poi funzioni API (funzioni di ingresso usate per orchestrare simulazioni/analisi)


MODELLO TEORICO
===============

1) Struttura Stackelberg
------------------------
A ogni periodo:
1. Il governo sceglie c_s (o mantiene quello corrente nel periodo MPC).
2. c_s viene tradotto in prescrizione sociale x_bar(c_s).
3. I cittadini osservano x_bar e il rischio p_t, poi scelgono x_t^*.
4. La dinamica SIR evolve usando x_t^* nel termine di contagio.

Il governo ottimizza anticipando la best response dei cittadini.

2) Mappa policy -> prescrizione
--------------------------------
Nel caso usato in simulazione:

x_bar(c_s) = 1 / (1 + kappa * c_s)

Proprieta:
- x_bar in [0,1]
- c_s piu alto -> x_bar piu basso (prescrizione piu restrittiva)

3) Rischio percepito
--------------------

p_t = clip(rho_rischio * I_t / N, 0, 1)

Quindi il rischio cresce con la quota di infetti.

4) Utility del cittadino e best response
----------------------------------------

4.1 Utility quadratica (best response chiusa)
U_q(x) = - lambda*p_t*x - (eta/2)*(x - x_bar)^2

Best response (nel codice con lambda=1 per default):
x_t^* = clip(x_bar - (lambda*p_t)/eta, 0, 1)

4.2 Utility logaritmica pura
U_l(x) = a*log(x/x_bar + epsilon) - lambda*p_t*x

Best response calcolata numericamente su griglia in [0,1].

5) Dinamica epidemiologica
--------------------------
Con fattore di contatto f(x)=x^potenza (in simulazione potenza=1):

new_infections_t = f(x_t^*) * beta * S_t * I_t / N
new_recoveries_t = gamma * I_t

S_{t+1} = S_t - new_infections_t
I_{t+1} = I_t + new_infections_t - new_recoveries_t
R_{t+1} = R_t + new_recoveries_t

6) Costo del governo su finestra MPC
------------------------------------

J(c_s) = J_epidemico(c_s) + J_controllo(c_s)

Dove:
- J_epidemico: somma dei costi istantanei previsti sulla finestra
- J_controllo = lambda_reg_controllo * c_s^2 * (orizzonte+1)

Costo istantaneo:
C_t = c_i*I_t + k*(exp(alpha*overload_t)-1)

Nel modello attuale il controllo entra soprattutto:
- indirettamente (modificando I_t tramite comportamento)
- direttamente con la penalita quadratica su c_s.


PARAMETRI DEL MODELLO (SIGNIFICATO, SETTING, MOTIVAZIONE)
==========================================================

A) Parametri epidemiologici
---------------------------
- N = 10000
  Popolazione totale. Scala del sistema.
- I_t0 = 10, R_t0 = 0, S_t0 = N-I_t0-R_t0
  Condizioni iniziali epidemiche.
- beta = 0.3
  Tasso trasmissione base.
- gamma = 0.05
  Tasso guarigione.
- T = 1000
  Durata totale della simulazione.
- t = np.arange(0, T + 1)
  Griglia temporale usata per output e plotting.
- R0 = beta/gamma = 6.0
  Numero riproduttivo base implicito.

B) Parametri costo sanitario/policy
-----------------------------------
- costo_infetto_giornaliero = 10.0
  Peso lineare degli infetti nel costo.
- k_saturazione_ospedali = 10.0
  Scala del termine non lineare di saturazione ospedaliera.
- alpha_saturazione_ospedali = 0.01
  Pendenza esponenziale della saturazione.
- capacita_ospedaliera_infetti = 250.0
  Soglia di capacita per overload.
- m_controllo = 2.0
  Parametro legacy del vecchio modello, mantenuto per retro-compatibilita ma non centrale nel flusso Stackelberg corrente.
- lambda_reg_controllo = 14.0
  Penalizzazione quadratica della policy c_s.

Scelta di modellazione attuale:
- il termine di costo proporzionale ai suscettibili e stato rimosso per mantenere il modello piu semplice;
- il controllo iniziale nelle simulazioni e fissato al valore minimo ammesso (`c_iniziale_default = c_min_default`).

Motivazione esplicita nel codice:
- k e alpha sono ridotti rispetto a versioni precedenti per evitare overflow numerico.

C) Parametri MPC e trigger
--------------------------
- intervallo_controllo_default = 80
  Ogni quanti giorni il governo ri-ottimizza.
- orizzonte_predizione_default = 80
  Lunghezza finestra predittiva per ogni ottimizzazione MPC.
- c_min_default = 0.05
  Controllo minimo (mai 0 pieno, ma molto basso).
- c_max_default = 80.0
  Controllo massimo ammissibile.
- c_iniziale_default = c_min_default
  Valore iniziale del controllo nelle simulazioni standard.
- soglia_attivazione_controllo_default = 0.005
  Attivazione controllo se I_t >= 0.5% della popolazione.
- fattore_isteresi_default = 0.3
  Soglia di spegnimento = 30% della soglia di attivazione.
- verbose_progress_default = True
  Attiva la stampa del progresso della simulazione.
- mostra_grafici_default = True
  Se True, le figure vengono mostrate con plt.show(); altrimenti vengono solo salvate e chiuse.
- num_grid_points_default = 401
  Granularita grid-search su c_s.
- step_percent_progress_default = 5
  Passo percentuale usato per i messaggi di avanzamento.

D) Parametri di regime e assetto
--------------------------------
- assetto_prudente_default = True
  Se True mantiene spente le scansioni di calibrazione e usa l'assetto prudente corrente.
- esegui_scansione_parametri_default = False
  Flag scansione su alpha/lambda.
- esegui_scansione_trigger_default = False
  Flag scansione su soglia/isteresi.
- esegui_scansione_comportamento_default = False
  Flag scansione su kappa/eta/rho.
- esegui_scansione_target_picco_default = False
  Flag scansione su target di picco.

E) Parametri comportamentali
----------------------------
- kappa_prescrizione = 0.05
  Sensibilita prescrizione rispetto a c_s.
- target_socialita_al_massimo_controllo = 1 / (1 + kappa_prescrizione * c_max_default)
  Socialita prescritta implicita quando il controllo e al massimo ammesso.

DESCRIZIONE DELLE FUNZIONI PER GRUPPI (HELPER -> API)
======================================================

3) Layer follower: comportamento cittadini
-----------------------------------------
3A) Mappatura policy/prescrizione + utility
- Helper:
  - utilita_cittadino_logaritmica(...)
    Ruolo: payoff follower in forma logaritmica pura.
- API:
  - socialita_prescritta_da_governo(c_s, kappa, tipo="logistica")
    Ruolo: trasforma la policy economica c_s in prescrizione sociale x_bar.
  - rischio_percepito(I_t, N, rho_rischio=1.0)
    Ruolo: trasforma lo stato epidemico in rischio percepito p_t.

3B) Best response dei cittadini
- API:
  - best_response_cittadino_quadratica(...)
    Ruolo: best response in forma chiusa (con clipping in [0,1]).
  - best_response_cittadino_logaritmica(...)
    Ruolo: best response numerica per utility logaritmica pura.

3C) Utility numeriche di supporto
- Helper:
  - fattore_contatto_da_socialita(x_t, potenza=1.0)
    Ruolo: converte socialita in moltiplicatore del contagio.
  - stampa_progresso_simulazione(...)
    Ruolo: monitoraggio avanzamento simulazione.
  - exp_sicura_scalata(argomento, scala=1.0)
    Ruolo: exp robusta con clipping anti-overflow.
  - somma_sicura(valori)
    Ruolo: somma con saturazione numerica.
  - overload_ospedaliero(I_t, capacita_ospedaliera)
    Ruolo: overload ospedaliero normalizzato.
  - costruisci_griglia_controllo(c_min, c_max, num_punti)
    Ruolo: griglia lineare/geometrica per grid search su c_s.

4) Funzioni di costo epidemiologico
-----------------------------------
- API:
  - calcola_costo_epidemiologico_istantaneo(...)
    Ruolo: costo epidemiologico giornaliero.
  - calcola_costo_epidemiologico_cumulato(...)
    Ruolo: aggregazione costi su intervallo temporale.
  - calcola_costo_epidemiologico_cumulato_con_controllo_variabile(...)
    Ruolo: costo cumulato con schedule di controllo variabile.

5) Dinamica Stackelberg e ottimizzazione leader
-----------------------------------------------
5A) Simulazione/costo su finestra
- API:
  - simula_finestra_predizione_stackelberg(...)
    Ruolo: simula traiettoria predetta a c_s costante.
  - costo_previsto_su_finestra_stackelberg(...)
    Ruolo: costruisce J(c_s) su finestra MPC.
  - ottimizza_c_s_su_finestra_stackelberg(...)
    Ruolo: minimizza J(c_s) con grid search.

5B) Simulazione completa MPC
- API:
  - simula_sir_stackelberg_con_controllo_periodico(...)
    Ruolo: simulatore end-to-end con trigger, isteresi e riottimizzazione periodica.

6) Analisi output
-----------------
6A) Visualizzazione
- API:
  - plot_dinamica_compartimenti_stackelberg(...)
    Ruolo: grafico compartimenti + incidenza.
    Nota: la barra "nuovi infetti" usa beta*S*I/N senza moltiplicatore socialita.
  - plot_controllo_e_comportamento_stackelberg(...)
    Ruolo: grafico diagnostico policy/comportamento/rischio.
    Nota: i pannelli nel plot sono 4.

6B) Scansioni e calibrazioni parametriche
- Helper:
  - valuta_scenario_target_picco(...)
    Ruolo: valutazione di una singola coppia (lambda, c_max) usata dalle scansioni target.
- API:
  - esegui_scansione_alpha_lambda(...)
  - esegui_scansione_trigger_isteresi(...)
  - esegui_scansione_comportamento(...)
  - calibra_parametri_logaritmica_min_picco_due_stadi(...)
  - esegui_scansione_target_picco(...)
  - esegui_scansione_target_picco_due_stadi(...)
    Ruolo comune: analisi sensibilita, calibrazione e benchmark su target di picco; funzioni disponibili ma non invocate nel main.


COSA SUCCEDE IN UNA RUN ATTUALE (PASSO-PASSO)
==============================================

1) Inizializzazione
- Import librerie.
- Set parametri globali.
- Calcolo R0=beta/gamma.

2) Definizione tipi confronto
- Utility quadratica: usa il setting corrente di eta_compliance e rho_rischio.
- Utility logaritmica: usa i default correnti (a, lambda_rischio, rho, epsilon).

3) Simulazione 1 (quadratica)
- Chiamata a simula_sir_stackelberg_con_controllo_periodico(..., tipo_best_response="quadratica").
- Produce traiettorie complete e log ottimizzazioni MPC.

4) Simulazione 2 (logaritmica)
- Chiamata a simula_sir_stackelberg_con_controllo_periodico(..., tipo_best_response="logaritmica").
- Usa i default logaritmici correnti.

5) Metriche finali
Per entrambe le simulazioni:
- picco infetti e giorno del picco
- costo totale epidemico
- compliance media x*/x_bar
- gap medio x*-x_bar

6) Confronto stampato
- Delta logaritmica - quadratica su:
  - picco
  - costo totale
  - compliance

7) Grafici salvati
- Quadratica:
  - sir_compartimenti_stackelberg_quadratica.png
  - sir_stackelberg_analisi_quadratica.png
- Logaritmica:
  - sir_compartimenti_stackelberg_logaritmica.png
  - sir_stackelberg_analisi_logaritmica.png

8) Visualizzazione
- plt.show() apre le figure.
